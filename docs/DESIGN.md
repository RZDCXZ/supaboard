## 1. 设计目标

SupaBoard 是用于学习 Supabase 的多人协作任务板。界面服务于 Auth、RLS、Storage 和 Realtime 等能力的演示，不追求复杂视觉效果。

设计遵循以下原则：

1. **业务优先**：页面层级直接对应工作区、任务、成员、评论、附件和活动记录，不增加无业务价值的页面。
2. **状态可见**：登录、保存、上传、实时连接、权限失败和空数据均有明确反馈。
3. **权限透明**：Owner 专属操作需要清晰标识，但最终权限仍由数据库 RLS、Storage policy 和服务端能力强制执行。
4. **桌面优先**：主要使用场景为开发者在桌面浏览器中学习和调试；窄屏保证基本可用，不做专项移动端体验。
5. **样式克制**：采用中性浅色后台风格，以绿色作为主要强调色，不引入独立品牌插画或复杂动效。

## 2. 信息架构

### 2.1 路由与页面职责

| 路由 | 页面结构 | 主要操作 |
| --- | --- | --- |
| `/login` | 居中 Auth 卡片 | 邮箱密码登录、GitHub OAuth、前往注册或密码恢复 |
| `/signup` | 居中 Auth 卡片 | 邮箱密码注册、展示邮箱确认提示 |
| `/forgot-password` | 居中 Auth 卡片 | 发送密码恢复邮件、返回登录 |
| `/update-password` | 居中 Auth 卡片 | 输入并确认新密码 |
| `/auth/callback` | 无常驻页面 | 交换 PKCE code；处理中显示全页加载，失败时显示错误和返回登录入口 |
| `/app` | 应用框架 + 工作区总览 | 查看、创建和进入工作区 |
| `/app/workspaces/[workspaceId]` | 应用框架 + 工作区详情 | 管理任务、成员、评论、附件，查看在线状态和活动记录 |
| `/app/settings` | 应用框架 + 设置表单 | 修改昵称、上传头像、退出登录 |

### 2.2 登录后导航层级

```text
SupaBoard
├── 工作区总览
├── 工作区
│   ├── 工作区 A
│   │   ├── 任务
│   │   ├── 成员
│   │   └── 活动
│   └── 工作区 B
└── 设置
```

左侧边栏负责全局导航与工作区切换；页面内页签只负责当前工作区的信息切换。任务详情通过右侧抽屉呈现，不新增任务详情路由。

## 3. 视觉规范

### 3.1 色彩

所有颜色均使用下表中的语义名称。实现时优先映射为 CSS 自定义属性，业务组件不直接散落十六进制颜色。

| 语义 | 色值 | 用途 |
| --- | --- | --- |
| `background` | `#F7F8FA` | 页面背景 |
| `surface` | `#FFFFFF` | 卡片、边栏、弹窗、抽屉 |
| `surface-subtle` | `#F1F3F5` | 次级区域、禁用背景、表头 |
| `text-primary` | `#18212F` | 标题、正文、主要数据 |
| `text-secondary` | `#5E6878` | 说明、时间、辅助标签 |
| `text-disabled` | `#98A1AE` | 禁用文字、占位提示 |
| `border` | `#DDE2E8` | 默认边框与分隔线 |
| `border-strong` | `#BBC3CE` | 悬停边框、强调分隔线 |
| `primary` | `#137A50` | 主按钮、选中项、链接、成功强调 |
| `primary-hover` | `#0F6843` | 主色悬停 |
| `primary-pressed` | `#0B5738` | 主色按下 |
| `primary-soft` | `#E8F5EE` | 选中背景、绿色标签背景 |
| `info` | `#2563EB` | 信息提示、连接中 |
| `info-soft` | `#EFF6FF` | 信息提示背景 |
| `warning` | `#B45309` | 警告、上传中、进行中 |
| `warning-soft` | `#FFF7E8` | 警告背景 |
| `danger` | `#B42318` | 删除、错误、断开连接 |
| `danger-hover` | `#912018` | 危险操作悬停 |
| `danger-soft` | `#FEF3F2` | 错误提示背景 |
| `focus` | `#0F766E` | 键盘焦点环 |

任务状态颜色：

| 状态 | 标签文字 | 标签背景 | 中文文案 |
| --- | --- | --- | --- |
| `todo` | `#475467` | `#F2F4F7` | 待办 |
| `in_progress` | `#B45309` | `#FFF7E8` | 进行中 |
| `done` | `#137A50` | `#E8F5EE` | 已完成 |

任务优先级颜色：

| 优先级 | 标签文字 | 标签背景 | 中文文案 |
| --- | --- | --- | --- |
| `low` | `#475467` | `#F2F4F7` | 低 |
| `medium` | `#B45309` | `#FFF7E8` | 中 |
| `high` | `#B42318` | `#FEF3F2` | 高 |

颜色不能作为唯一状态表达。状态与优先级始终同时显示中文文字；错误输入同时显示错误文案和边框状态。

### 3.2 字体

- 字体族：`ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。
- 正文字号：`14px`，行高 `22px`，字重 `400`。
- 辅助文字：`12px`，行高 `18px`，字重 `400`。
- 页面标题：`24px`，行高 `32px`，字重 `600`。
- 区块标题：`18px`，行高 `26px`，字重 `600`。
- 组件标题：`16px`，行高 `24px`，字重 `600`。
- 按钮和标签：`14px`，行高 `20px`，字重 `500`。
- 数字统计使用等宽数字特性 `font-variant-numeric: tabular-nums`，不单独引入字体。

### 3.3 间距、圆角与阴影

间距以 `4px` 为基础单位，只使用 `4、8、12、16、20、24、32、40、48px`。

| 项目 | 规格 |
| --- | --- |
| 页面左右留白 | 桌面 `32px`，平板 `24px`，窄屏 `16px` |
| 区块间距 | `24px` |
| 卡片内边距 | `20px`，紧凑卡片 `16px` |
| 表单字段间距 | `16px` |
| 标签与控件间距 | `8px` |
| 默认圆角 | `8px` |
| 小标签圆角 | `999px` |
| 弹窗与抽屉圆角 | `12px` |
| 默认阴影 | `0 1px 2px rgba(16, 24, 40, 0.06)` |
| 浮层阴影 | `0 12px 32px rgba(16, 24, 40, 0.14)` |

边框默认 `1px solid #DDE2E8`。不使用大面积渐变、玻璃拟态或多层装饰阴影。

### 3.4 图标与动效

- 图标使用同一套线性图标库，默认 `18px`，小型状态图标 `16px`。
- 图标按钮必须有可访问名称和 Tooltip，不能使用 Emoji 代替功能图标。
- 普通颜色、背景和边框过渡为 `150ms ease-out`。
- 弹窗与抽屉使用 `180ms ease-out` 的淡入和位移动画。
- 加载旋转动画保持匀速，并尊重 `prefers-reduced-motion`；减少动态时取消位移，只保留必要的透明度变化。

## 4. 全局应用框架

### 4.1 桌面布局

登录后的页面共享以下结构：

```text
┌────────────── 240px ──────────────┬──────────────────────────────┐
│ 品牌与总览入口                     │ 页面标题 / 页面级操作          │
│ 工作区列表                         ├──────────────────────────────┤
│                                    │                              │
│                                    │ 页面内容                      │
│                                    │                              │
│ 设置                               │                              │
│ 当前用户                           │                              │
└────────────────────────────────────┴──────────────────────────────┘
```

- 边栏宽度固定 `240px`，高度为视口高度，使用 `surface` 背景和右侧边框。
- 内容区最小宽度为 `0`，独立滚动；页面内容最大宽度 `1440px`，居中显示。
- 页面标题区最小高度 `72px`，标题左对齐，主要操作位于右侧。
- 边栏顶部显示文字标识 “SupaBoard”，不额外制作品牌插画。
- “工作区总览”位于工作区列表之前；工作区列表按最近访问排序，名称过长时单行省略。
- 当前路由使用 `primary-soft` 背景、`primary` 文字和左侧 `3px` 强调线。
- 边栏底部固定“设置”入口与当前用户头像、昵称；点击用户区域进入设置页。

### 4.2 边栏状态

- 工作区不超过 8 个时全部显示；超过 8 个时工作区区域内部滚动。
- 工作区加载时显示 3 条骨架项。
- 没有工作区时显示“还没有工作区”和“创建工作区”文本按钮。
- 用户资料加载失败时显示默认头像和“用户”，不显示邮箱。
- Owner 与 member 的差异不在边栏中强调，角色在工作区总览卡片和成员页展示。

### 4.3 基础响应式规则

| 视口宽度 | 行为 |
| --- | --- |
| `>= 1024px` | 固定显示 `240px` 边栏；任务详情为右侧 `480px` 抽屉 |
| `768px–1023px` | 边栏收起，通过顶部菜单按钮打开覆盖式导航；内容保持桌面信息结构 |
| `< 768px` | 页面单列；统计卡片横向滚动或两列换行；表格转为卡片列表；任务抽屉占满视口 |

窄屏只保证核心操作可完成，不重新设计移动端导航，不提供手势操作。

## 5. Auth 页面

### 5.1 通用布局

`/login`、`/signup`、`/forgot-password` 和 `/update-password` 共享 Auth 布局：

- 页面使用 `background`，内容在视口内水平、垂直居中。
- 顶部显示 “SupaBoard” 和一句短说明；不使用图片或左右分栏。
- 表单卡片宽度 `400px`、最大宽度 `calc(100vw - 32px)`，内边距 `32px`。
- 卡片标题与说明靠左；字段纵向排列；主按钮占满卡片宽度。
- 页面底部只展示当前流程需要的跳转链接，避免重复导航。

### 5.2 `/login`

从上到下包含：

1. 标题“登录 SupaBoard”和简短说明。
2. 邮箱、密码字段。
3. “忘记密码？”链接，位于密码标签同一行右侧。
4. 全宽主按钮“登录”。
5. 带“或”文案的分隔线。
6. 全宽次按钮“使用 GitHub 登录”。
7. 底部文案“还没有账号？注册”。

提交期间禁用全部提交入口，主按钮显示加载状态。凭据错误显示卡片内错误提示，不通过 Toast 隐藏关键错误。

### 5.3 `/signup`

包含邮箱、密码、确认密码和“注册”按钮。注册成功后用卡片内成功状态替换表单：

- 标题“请检查邮箱”。
- 说明确认邮件已发送，不声称用户已经登录。
- 提供“返回登录”入口；邮件重新发送流程不在首版范围内。

### 5.4 `/forgot-password`

包含邮箱字段和“发送重置邮件”按钮。成功后保留用户提交的脱敏邮箱提示，并提供返回登录入口。无论邮箱是否存在，界面使用相同成功文案，避免暴露账号存在性。

### 5.5 `/update-password`

包含新密码、确认新密码和“更新密码”按钮。恢复会话无效时不显示表单，展示“恢复链接无效或已过期”及“重新发送恢复邮件”入口。

### 5.6 `/auth/callback`

- 默认显示居中加载指示器与“正在完成登录…”文案。
- 成功后直接跳转，不显示成功页。
- code exchange 失败时显示错误卡片，提供“返回登录”按钮；不展示原始错误、code 或 token。

## 6. 工作区总览 `/app`

### 6.1 页面结构

```text
页面标题“工作区”                         [创建工作区]
说明文字

工作区卡片列表
┌──────────────────────────────────────────────────────┐
│ 工作区名称                 Owner / 成员    更新时间   │
└──────────────────────────────────────────────────────┘
```

- 桌面端使用单列列表，最大宽度 `960px`，便于比较角色和更新时间。
- 每个工作区项最小高度 `76px`，整行可点击；名称下方可显示成员数量（查询结果存在时）。
- 角色使用标签显示为“Owner”或“成员”，不翻译数据库中的原始 role 值后再参与授权判断。
- 更新时间显示相对时间，Tooltip 或可访问文本提供完整时间。
- 默认按更新时间倒序；列表首项作为最近更新的工作区入口，点击进入工作区页面。

### 6.2 创建工作区弹窗

- 由页面右上角“创建工作区”主按钮打开。
- 弹窗宽度 `440px`，标题“创建工作区”，只包含名称字段；名称去除首尾空白后为 `1～100` 个字符。
- 底部操作为“取消”和“创建”；创建期间禁止关闭和重复提交。
- 成功后关闭弹窗并进入新工作区。
- 校验错误显示在字段下方；服务端失败保留输入并显示弹窗内错误提示。

### 6.3 页面状态

- 首屏加载：标题保留，列表显示 3 条骨架项。
- 空状态：显示“还没有工作区”，说明创建后可以添加任务和成员，并提供主按钮。
- 加载失败：显示内联错误和“重试”按钮。
- 创建冲突或未知错误：使用稳定业务文案，不展示数据库错误详情。

## 7. 工作区详情 `/app/workspaces/[workspaceId]`

### 7.1 页面头部

页面头部包含：

- 返回工作区总览的面包屑。
- 工作区名称；只有 Owner 显示重命名入口。
- 当前用户角色标签。
- 在线成员头像组，最多直接显示 5 人，超出部分显示 `+N`。
- Realtime 状态：正常时不额外占用空间；连接中、重连或断开时显示状态条。

头部下方固定三个页签：“任务”“成员”“活动”。页签状态保留在 URL 查询参数中，例如 `?tab=members`；默认进入“任务”。

### 7.2 任务页签

页面从上到下包含统计、筛选工具栏和任务分组列表。

#### 统计区

显示 4 个统计卡片：全部、待办、进行中、已完成。

- 桌面端四列等宽排列，卡片最小高度 `88px`。
- 卡片显示中文名称和数量，数量字号 `24px`。
- 统计卡片只展示信息，不作为筛选按钮，避免与下方筛选器产生两套状态。
- 统计加载失败不阻断任务列表，单独显示“统计暂不可用”。

#### 筛选与操作区

从左到右包含：

1. 状态选择器：全部、待办、进行中、已完成。
2. 负责人选择器：全部、未分配、具体成员。
3. “清除筛选”文本按钮，仅在存在筛选时显示。
4. 右侧主按钮“新建任务”。

筛选写入 URL 查询参数，刷新后保持。切换筛选时回到第一页。任务每页 20 条，底部提供上一页、下一页和当前范围；不允许用户将每页数量设置超过 100。

#### 任务分组列表

- 默认按 `todo`、`in_progress`、`done` 顺序显示三个分组，不实现拖拽。
- 使用文字标题和数量显示分组状态，分组可折叠但默认展开。
- 同组任务按更新时间倒序。
- 单个任务项包含标题、优先级、负责人头像与昵称、评论数、附件数和更新时间。
- 未分配负责人显示“未分配”，不显示空头像。
- 点击任务项打开详情抽屉；键盘 Enter 或 Space 具有相同行为。
- Realtime 新增任务插入正确分组；状态变化时移动到目标分组；删除时从列表移除。
- 当前筛选无法容纳新事件时不强行插入，刷新当前页数据并更新统计。

#### 新建任务弹窗

弹窗宽度 `560px`，包含标题、描述、状态、优先级和负责人。标题去除首尾空白后为 `1～200` 个字符，描述最多 `5000` 个字符。状态默认“待办”，优先级默认“中”，负责人默认“未分配”。

只有标题为必填项。创建期间禁用提交按钮；成功后关闭弹窗并由当前数据更新策略展示任务。

### 7.3 任务详情抽屉

桌面端抽屉从右侧覆盖内容，宽度 `480px`、最大宽度 `100vw`，内部独立滚动；窄屏占满视口。打开时 URL 增加 `task` 查询参数，关闭时移除，便于刷新恢复当前任务上下文，但不新增路由。

抽屉结构：

1. 顶部：任务标题、状态标签、关闭按钮和更多操作菜单。
2. 任务属性：状态、优先级、负责人，可直接编辑。
3. 描述：查看与编辑模式。
4. 附件区：上传入口、文件列表、下载和允许的删除操作。
5. 评论区：时间顺序评论列表、删除操作、输入状态和评论表单。

编辑规则：

- 标题、描述、状态、优先级和负责人保存时显示字段级进行中状态。
- 保存失败恢复最后一次服务端值并显示内联错误；不以未确认的乐观结果覆盖服务端事实。
- 负责人下拉只包含当前工作区成员。
- 删除任务位于更多菜单，点击后打开危险确认弹窗。
- Realtime 收到当前任务更新时同步字段；用户正在编辑同一字段时显示“任务已被其他成员更新”，要求重新加载后再编辑，避免静默覆盖。

#### 评论区

- 评论显示作者头像、昵称、正文和时间。
- 评论作者显示删除入口；Owner 对全部评论显示删除入口；其他成员无删除入口。
- 输入框支持多行，显示 `当前字数 / 2000`，空白内容不可提交。
- 其他成员输入时，在输入框上方显示“某某正在输入…”。多个成员同时输入时最多展示两个昵称，更多人使用“等 N 人”。
- Realtime 断开时隐藏输入状态，并在表单旁提示“实时连接已断开，评论提交仍会尝试保存”。

#### 附件区

- 上传入口支持文件选择，不在首版设计拖拽上传；支持常见图片、PDF 和纯文本文件。
- 选择文件后先校验类型和 `10 MB` 上限，再显示上传进度。
- 文件项显示文件名、格式、大小和上传者；下载时展示短暂进行中状态。
- 上传者或 Owner 显示删除入口，其他成员只显示下载。
- 下载前生成有效期 `60 秒` 的签名 URL；生成失败时显示“下载链接生成失败，请重试”，不展示对象路径。
- 对象上传成功但元数据写入失败时提示上传未完成；补偿删除由应用处理，界面不声称附件已经保存。

### 7.4 成员页签

页面头部显示成员数量。Owner 看到“添加成员”主按钮，普通成员不显示成员管理按钮。

成员列表包含头像、昵称、角色和加入时间，不显示邮箱。Owner 成员始终置顶，其余按加入时间排序。

Owner 操作：

- “添加成员”打开 `440px` 弹窗，仅包含邮箱字段。
- 提交后根据 Edge Function 状态显示参数错误、未登录、非 Owner、用户不存在或已是成员等明确提示。
- 普通成员行显示“移除”入口；Owner 自己的行不显示移除或降级入口。
- 移除成员必须经过危险确认弹窗，确认文案说明对方后续请求和重新连接将失去权限。

普通成员直接访问成员页签时可查看列表，但不能通过界面管理成员；服务端与数据库仍必须独立校验权限。

### 7.5 活动页签

- 使用垂直时间线展示任务新增、状态变更和删除记录。
- 每项包含操作者、动作、任务标题或可用标识以及时间。
- `actor_id` 为空时显示“系统”；已删除任务不提供不可用链接。
- 活动记录为只读，不显示新增、编辑或删除操作。
- 默认按时间倒序，每页 20 条，通过“加载更多”追加。

### 7.6 工作区页面异常状态

| 状态 | 界面行为 |
| --- | --- |
| 工作区不存在 | 显示“工作区不存在”及返回总览入口 |
| 非工作区成员 | 显示“无权访问此工作区”，不透露工作区名称、成员或任务数据 |
| 成员被移除 | 清除当前工作区数据，停止订阅并返回工作区总览，显示权限变化提示 |
| Realtime 连接中 | 页面顶部显示蓝色状态条“正在连接实时更新…” |
| Realtime 断开 | 显示红色状态条与“重新连接”按钮，现有服务端操作仍可继续尝试 |
| Realtime 恢复 | 重新获取当前页面数据后显示短暂成功 Toast |
| 首屏部分查询失败 | 保留成功区块，失败区块独立显示错误和重试入口 |
| 分页结果为空 | 显示当前筛选无结果，并提供清除筛选入口 |

## 8. 设置页 `/app/settings`

### 8.1 页面结构

内容最大宽度 `720px`，分为“个人资料”和“账号”两个卡片区块。

个人资料包含：

- 当前头像、昵称和更换头像按钮。
- 昵称输入框，去除首尾空白后为 `1～80` 个字符，并显示长度提示与保存按钮。
- 头像文件要求说明：JPEG、PNG 或 WebP，不超过 `2 MB`。
- 上传过程显示进度；成功后立即刷新公共头像 URL；失败时保留原头像。

账号区包含退出登录按钮。退出不是危险数据操作，不使用红色实心按钮；使用边框按钮并在提交期间显示进行中状态。成功后返回 `/login`。

### 8.2 头像状态

- 无头像时显示由昵称首字符生成的中性底色头像。
- 文件类型或大小不合法时，在选择入口下方显示错误，不发起上传。
- upsert 失败时保留当前头像并允许重试。
- 页面不展示或编辑邮箱，因为 profile 设计只对协作成员公开昵称与头像。

## 9. 组件规格

### 9.1 按钮

| 尺寸 | 高度 | 水平内边距 | 使用场景 |
| --- | --- | --- | --- |
| 小 | `32px` | `12px` | 表格行、次级操作 |
| 默认 | `40px` | `16px` | 表单和页面操作 |
| 大 | `44px` | `18px` | Auth 主操作 |

按钮类型：

- 主按钮：`primary` 背景、白色文字，每个区域最多一个主要操作。
- 次按钮：白色背景、默认边框、主要文字。
- 文本按钮：透明背景，用于低强调操作和链接式动作。
- 危险按钮：仅用于最终确认删除或移除，使用 `danger` 背景和白色文字。
- 图标按钮：最小点击区域 `40 × 40px`，即使图标只有 `18px`。

加载时保留原宽度，显示 Spinner 并禁用重复点击。禁用状态不能只降低透明度，还要取消指针反馈并提供明确的 disabled 属性。

### 9.2 表单控件

- 输入框、选择器默认高度 `40px`，多行输入最小高度 `96px`。
- 标签位于控件上方，间距 `6px`；必填项使用“必填”文本或星号并提供可访问说明。
- 说明和错误位于控件下方，字号 `12px`。
- 错误边框使用 `danger`，同时设置 `aria-invalid` 和错误描述关联。
- 密码字段提供显示或隐藏按钮，按钮具备可访问名称。
- 表单提交失败后焦点移动到首个错误字段；非字段错误聚焦到错误摘要。

### 9.3 标签与头像

- 状态、优先级和角色标签高度 `24px`，水平内边距 `8px`，圆角 `999px`。
- 角色标签只使用“Owner”“成员”两种显示值。
- 默认头像尺寸 `32px`，成员列表使用 `40px`，设置页使用 `72px`。
- 头像组重叠 `8px`，每个头像保留 `2px` 白色描边。

### 9.4 卡片与列表

- 卡片使用白色背景、默认边框和 `8px` 圆角；普通静态卡片不使用浮层阴影。
- 可点击列表项悬停使用 `surface-subtle`，键盘焦点使用统一焦点环。
- 列表分隔优先使用单条底部边框，避免每行形成过多独立卡片。
- 骨架屏形状对应最终内容，不使用整页通用 Spinner 替代结构化加载。

### 9.5 弹窗与确认框

- 普通弹窗宽度为 `440px` 或 `560px`，窄屏宽度为 `calc(100vw - 32px)`。
- 标题、说明、内容和底部操作区边界清晰；操作区按钮右对齐。
- 点击遮罩或 Escape 可关闭未提交的普通弹窗。
- 提交中禁止关闭，避免用户误判操作结果。
- 删除任务、移除成员和删除附件使用确认框；必须写明对象名称和后果，最终按钮使用明确动词。

### 9.6 Toast 与内联反馈

- Toast 位于右上角，宽度 `360px`，最多同时显示 3 条。
- 成功 Toast 自动关闭时间为 4 秒；错误 Toast 默认不自动关闭。
- 表单验证、Auth 失败和需要用户立即处理的问题使用内联反馈，不只使用 Toast。
- Realtime 状态使用页面级状态条，不使用持续重复 Toast。
- 用户提示不包含 SQL、policy 名称、堆栈、JWT、对象路径或原始 Supabase 错误。

### 9.7 抽屉

- 桌面宽度 `480px`，顶部栏固定，内容区滚动。
- 打开后焦点移入抽屉标题或第一个可操作元素，背景内容不可获得焦点。
- Escape 关闭抽屉；存在未保存编辑时先显示离开确认。
- 关闭后焦点返回触发它的任务项。

## 10. 权限与安全在界面中的表达

- 未登录用户不能看到应用框架；访问 `/app/**` 时由服务端重定向。
- Owner 专属按钮只对 Owner 显示，但隐藏按钮不替代服务端、RLS 或 Edge Function 权限检查。
- 普通成员可以维护任务、评论和附件，但不能看到添加或移除成员入口。
- 评论删除入口依据“作者或 Owner”显示；附件删除入口依据“上传者或 Owner”显示。
- 非成员错误页不展示受保护资源名称，避免通过错误页面泄露工作区信息。
- 前端环境说明和错误信息不得出现 secret/service key；publishable key 也不在页面文案或调试区展示。
- 被移除成员收到权限错误后，客户端清理工作区缓存并取消相关 Realtime channel。

## 11. 加载、空状态与错误原则

### 11.1 加载

- 首屏查询使用与最终布局一致的骨架屏。
- 按钮操作使用按钮内加载状态，不阻塞无关页面区域。
- Realtime 更新不显示全页加载；需要重新获取时仅在对应区块显示轻量状态。
- 文件上传显示确定进度；无法获取精确进度时显示不确定进度条和“正在上传”。

### 11.2 空状态

| 场景 | 文案与操作 |
| --- | --- |
| 无工作区 | “还没有工作区” + “创建工作区” |
| 无任务 | “当前工作区还没有任务” + “新建任务” |
| 筛选无结果 | “没有符合当前筛选条件的任务” + “清除筛选” |
| 无评论 | “还没有评论，开始讨论吧” |
| 无附件 | “还没有附件” + 文件限制说明 |
| 无活动 | “还没有任务活动” |
| 只有 Owner | “还没有其他成员” + Owner 可见的“添加成员” |

### 11.3 错误

- 可恢复错误必须提供“重试”或可重复操作入口。
- 权限错误与网络错误使用不同文案，不能统一显示“系统错误”。
- 用户输入和上传失败后保留可安全保留的数据。
- 破坏性操作失败时不提前从界面移除对象；以服务端结果为准。
- 未知错误使用稳定文案并记录服务端 request ID；只在用户联系排障有价值时展示该 ID。

## 12. 可访问性

- 所有功能可通过键盘完成，Tab 顺序与视觉顺序一致。
- 可交互元素使用原生 `button`、`a`、`input`、`select` 等语义元素，不用普通 `div` 模拟。
- 键盘焦点环为 `2px solid #0F766E`，外偏移 `2px`，不能被全局样式移除。
- 正文和背景、按钮文字和按钮背景满足 WCAG AA 对比度要求；弱化文字不用于关键操作或错误。
- 所有字段有可见标签；占位符不能替代标签。
- 图标按钮、头像组、状态点和加载图标提供可访问名称或隐藏冗余描述。
- 弹窗与抽屉使用正确的 dialog 语义、焦点锁定和焦点恢复。
- Toast、保存结果和 Realtime 状态通过合适的 live region 播报；输入字数变化不频繁打断屏幕阅读器。
- 状态、优先级、连接状态和错误不能只依赖颜色表达。
- 点击区域不小于 `40 × 40px`；正文缩放到 200% 时仍能完成核心流程。

## 13. 文案规范

- 使用简洁、直接的中文动作，例如“创建工作区”“保存更改”“移除成员”。
- 数据库枚举只用于程序内部，界面显示为“待办”“进行中”“已完成”和“低”“中”“高”。
- 删除确认必须说明对象和影响，不使用含糊的“确定吗？”。
- Auth、RLS、Storage 或 Realtime 的底层错误转换为用户能采取行动的业务文案。
- 不向普通用户展示 Supabase policy 名称、SQL 错误、token、内部表名和 Storage 对象路径。
- 相对时间用于快速浏览，完整时间使用本地时区并通过 Tooltip 或辅助文本提供。

## 14. 实现边界与验收

本设计不改变 [`docs/PRD.md`](./PRD.md) 和 [`docs/TECH.md`](./TECH.md) 中的路由、数据模型、API、权限与文件限制。实现时不得因界面方便放宽 RLS、Storage policy 或 Edge Function 权限。

不在本设计范围内：

- 拖拽看板和任务排序。
- 独立设计系统包或通用组件站点。
- 富文本编辑、全文搜索和通知中心。
- 专项移动端导航与手势交互。
- 新增路由、数据库表、字段、API 或角色。
- 暗色模式和用户自定义主题。

页面实现至少满足以下验收条件：

1. 八个既定路由都有明确的页面或过渡状态。
2. Owner、成员、访客和非成员看到的操作与 PRD 权限一致。
3. 任务的创建、筛选、分页、编辑、评论、附件和删除均有加载、成功与失败状态。
4. 工作区任务和评论变化、在线成员、输入状态及连接异常均有可理解的实时反馈。
5. 头像与附件界面遵守格式、大小、公开性和删除权限约束。
6. 桌面端布局完整，窄屏不出现无法访问的核心操作或水平页面溢出。
7. 键盘、焦点、标签、错误关联和颜色对比符合本文档的基础无障碍要求。
